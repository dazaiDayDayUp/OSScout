# CNCF 项目成熟度评估标准

## 概述

Cloud Native Computing Foundation（CNCF）将开源项目分为三个成熟度等级：Sandbox（沙盒）、Incubating（孵化）、Graduated（毕业）。每个等级都有明确的进入和维持标准，是评估云原生开源项目健康度的重要参考框架。

## 三个成熟度等级

### Sandbox（沙盒）

- **进入门槛**：项目被 CNCF 技术监督委员会（TOC）接受
- **要求**：
  - 开源许可证（Apache-2.0 优先）
  - 至少 2 名来自不同组织的维护者
  - 基本的治理文档（README、CONTRIBUTING）
- **意义**：项目处于早期探索阶段，技术和社区尚未成熟

### Incubating（孵化）

- **进入门槛**：项目已在生产环境被至少 3 个独立组织使用
- **要求**：
  - 文档完整（用户文档、开发文档、API 文档）
  - 有明确的安全披露流程
  - 有公开的路线图
  - 至少 3 个来自不同组织的 committer
  - 有代码审查和 CI/CD 流程
  - 发布了至少一个稳定版本
- **意义**：项目已证明其在真实场景中的价值，社区开始多元化

### Graduated（毕业）

- **进入门槛**：项目在多个行业中广泛采用，社区高度成熟
- **核心要求**：
  - **采用度**：被至少 5 个不同行业的组织在生产环境使用
  - **贡献者多样性**：committer 来自至少 5 个不同组织
  - **治理成熟**：有正式的治理委员会（如 TSC、Maintainer 委员会）
  - **安全审计**：通过第三方安全审计
  - **版本管理**：遵循语义化版本（SemVer），有清晰的发布节奏
  - **兼容性承诺**：大版本间有明确的兼容性保证
  - **社区健康**：有活跃的用户和开发者社区（会议、Slack、邮件列表）

## 评估维度映射

| CNCF 标准 | 对应 OSScout 维度 | 关键指标 |
|----------|------------------|---------|
| 贡献者多样性 | 社区健康 | 来自 N 个不同组织的 committer |
| 采用度 | 社区健康 + 技术演进 | 生产环境采用组织数 |
| 安全审计 | 安全 | 第三方安全审计报告 |
| 版本管理 | 技术演进 | SemVer 遵循度、发布频率 |
| 治理成熟 | 社区健康 | 治理委员会、决策流程 |
| 文档完整 | 代码质量 | 文档覆盖度 |

## 对非云原生项目的适用性

CNCF 标准主要针对云原生基础设施项目，但其核心思想（贡献者多样性、采用度、治理成熟度）对任何类型的开源项目都有参考价值。

例如：
- 前端框架可以借鉴"采用度"指标（有多少知名项目在使用）
- 工具库可以借鉴"版本管理"和"兼容性承诺"
- 所有项目都应该关注"贡献者多样性"这一核心健康指标
