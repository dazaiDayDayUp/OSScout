# 前端框架竞品映射

## 领域概述

前端框架是 Web 应用开发的核心技术选型之一。本竞品映射覆盖 React、Vue、Svelte 三个主流框架，从社区健康、技术演进、生态成熟度等维度进行对比。

## React（Meta/Facebook）

- **定位**：声明式 UI 库，生态系统最丰富
- **GitHub**: facebook/react
- **社区规模**：
  - Stars: 230k+
  - 贡献者: 1900+
  - npm 周下载: ~2500 万次
- **技术特点**：
  - 虚拟 DOM + Fiber 架构
  - Hooks API（函数式组件为主）
  - React Server Components（RSC）
- **发布节奏**：平均每年 1-2 个 minor 版本
- **优势**：生态最丰富、就业市场需求最大、Meta 长期投入
- **风险**：API 变化较快（Class -> Hooks -> RSC），学习曲线陡峭

## Vue.js（Evan You / 独立）

- **定位**：渐进式框架，易学易用
- **GitHub**: vuejs/core
- **社区规模**：
  - Stars: 210k+
  - 贡献者: 500+
  - npm 周下载: ~600 万次
- **技术特点**：
  - 响应式系统（Proxy-based）
  - 组合式 API（Composition API）
  - 单文件组件（SFC）
- **发布节奏**：Vue 3 后约每 3-6 个月一个 minor 版本
- **优势**：学习曲线平缓、文档优秀、中文社区活跃
- **风险**：核心维护团队较小（~10 人），Evan You 个人影响力过大

## Svelte（Rich Harris / Vercel）

- **定位**：编译时框架，无虚拟 DOM
- **GitHub**: sveltejs/svelte
- **社区规模**：
  - Stars: 82k+
  - 贡献者: 800+
  - npm 周下载: ~100 万次
- **技术特点**：
  - 编译时优化，运行时极轻量
  - 真正的响应式（赋值即更新）
  - SvelteKit 全栈框架
- **发布节奏**：Svelte 5 大版本更新间隔较长（~2 年）
- **优势**：性能优异、bundle 体积小、代码简洁
- **风险**：社区规模较小、企业采用度有限、Svelte 5 的 Runes 引入 breaking change

## 选型建议

- **新项目**：React（生态最广）或 Svelte（性能最优）
- **团队经验有限**：Vue（学习成本最低）
- **长期维护**：React（Meta 背书，社区最活跃）
- **性能敏感**：Svelte（bundle 最小，运行时开销最低）
