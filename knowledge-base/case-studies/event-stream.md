# event-stream 供应链攻击 —— 潜伏两个月的恶意代码

## 背景

2018 年 9 月，npm 包 `event-stream`（一个流行的 Node.js 流处理工具，每周下载量约 200 万次）被发现包含恶意代码。该包的原维护者 dominictarr 因精力有限，将维护权转让给了一位 GitHub 上名为 "right9ctrl" 的陌生用户。

该用户在获得权限后，向 `event-stream` 中注入了一个针对特定加密货币钱包（Copay）的后门。恶意代码在 v3.3.6 中被引入，潜伏了约两个月才被安全研究员发现。

## 早期征兆

- **维护权转移给陌生人**： dominictarr 在 issue 中表示"我不再维护这个项目了，谁想接手？"，随后将权限转让给了一个几乎无贡献历史的 GitHub 账户
- **无 vetting 流程**：维护权转让没有任何背景调查或社区讨论
- **可疑的 commit**：right9ctrl 的首次 commit 就引入了对 `flatmap-stream` 的新依赖，该依赖是攻击的载体
- **目标性极强的 payload**：恶意代码仅在特定条件（Copay 应用内部）下激活，常规测试难以发现

## 影响

- Copay 用户（比特币钱包应用）的私钥可能被盗
- event-stream 被数万项目依赖，恐慌性升级席卷 npm 生态
- dominictarr 公开表示歉意，但强调"我没有义务维护这个项目"
- npm 随后加强了包维护权变更的监控

## 教训

1. **维护权转让是高风险事件**：任何维护者变更都应引起警觉，尤其是转让给无贡献历史的陌生人
2. **新增依赖需要审查**：即使是 patch 版本更新引入的新依赖也需要审查
3. **供应链纵深防御**：重要项目不应直接依赖流行包，应有自己的依赖审计流程
4. **维护者退出声明**："我不再维护了"的公开声明往往伴随着权力真空期的风险
5. **定向攻击更难发现**：与破坏式攻击（如 colors.js）不同，定向窃取型攻击更隐蔽、危害更大

## 相关指标

- Bus Factor: 从 1 变为 0（转让给陌生人）
- 恶意代码潜伏期: ~2 个月
- 每周下载量: ~200 万次
- 发现方式: 社区安全研究员的依赖审计
