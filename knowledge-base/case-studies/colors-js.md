# colors.js 恶意注入事件（详见 faker-js.md 合并分析）

本案例与 Faker.js 事件为同一维护者（Marak Squires）同时发起的攻击，详细信息请参考同目录下的 `faker-js.md`。

此处单独归档以便于按技术维度检索（"依赖漏洞"、"供应链安全"等关键词）。

## 技术要点补充

- 恶意版本号: `colors@1.4.1` 和 `colors@1.4.44-liberty-2`
- 攻击手法: 在 `lib/index.js` 中注入无限循环，使调用 `colors` 的进程 CPU 占满
- 检测难度: 代码经过混淆，不易在代码审查中发现
- 修复方式: 社区迅速回滚到 `1.4.0`，npm 撤销恶意版本
